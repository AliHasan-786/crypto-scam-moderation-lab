# Security and Red-Team Reporting

This lab welcomes good-faith reports about policy-boundary failures, false
positives, false negatives, prompt injection, evidence fabrication, privacy
issues, and operational-safety gaps.

## Scope

In scope: the public demo, deterministic scorer, evidence extractors, authored
evaluation fixtures, and future shadow-service code. Out of scope: targeting
real Bluesky users, mass collection, real-fraud evasion, or service disruption.

## Report

Use a private GitHub security advisory at
`https://github.com/AliHasan-786/crypto-scam-moderation-lab/security/advisories/new`.
Include a minimal sanitized reproduction and expected vs. observed result. Do
not include real user content, identifiers, wallets, or credentials.
