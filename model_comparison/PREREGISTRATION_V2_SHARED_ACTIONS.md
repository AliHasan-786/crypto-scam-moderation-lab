# Guard-Model Benchmark v2 Preregistration: Shared Actions

**Status:** frozen design; no v2 external request has been made.

## 1. Motivation

The v1 hosted comparison used a two-action hosted mapping against a three-action
baseline. Its six-case skeptical-reportage result is valuable but cannot support
a like-for-like action-tier comparison. V2 fixes that interface before any
additional collection.

## 2. Frozen Inputs and Systems

- Dataset: the same hash-pinned 168-row `test.csv`; no rows added, removed, or
  relabeled after results are seen.
- Targeted slice: the six hash-pinned skeptical-reportage cases in
  `documented_skeptical_reportage_slice.json`; report `n=6` on every result.
- Systems: local TF-IDF baseline, `meta-llama/llama-guard-4-12b`, and
  `openai/gpt-oss-safeguard-20b`, all evaluated through one shared adapter.
- Inputs: original post text only. No retrieval, account metadata, URL fetches,
  or platform history.

## 3. Shared Three-Action Contract

Every system must produce, or be adapted into, exactly this schema:

```json
{
  "action": "public_label_candidate | human_review | no_action",
  "speaker_originated_evidence": ["..."],
  "risk_indicators": ["..."],
  "protected_context": ["warning | reportage | satire | research | debugging | help_seeking"],
  "rationale": "short policy-grounded explanation"
}
```

Adapter rules, applied identically after output validation:

1. `public_label_candidate` requires at least two policy-defined risk
   indicators, at least one speaker-originated ask or promotion, no protected
   context, and a nonempty supporting-evidence list. Otherwise downgrade to
   `human_review`.
2. `human_review` is for a plausible policy risk with missing or conflicting
   evidence, including a protected context paired with an unresolved
   speaker-originated ask or promotion.
3. `no_action` is for no plausible crypto-investment-scam evidence, including a
   protected context without a speaker-originated ask or promotion.
4. Invalid JSON, missing fields, or unsupported categories are `incompatible`,
   counted and published; they never become `no_action`.
5. The baseline uses its frozen score thresholds as its adapter: score >= label
   threshold -> public-label candidate; review <= score < label -> human review;
   otherwise no action. Its evidence fields are generated by the same
   deterministic evidence extractor used elsewhere in the lab.

## 4. Hypotheses and Metrics

**H1 (primary):** under the shared adapter, external systems have a lower
protected-context escalation rate (`public_label_candidate` or `human_review`)
than the baseline on the documented six-case slice. This direction is informed
by v1 and is explicitly directional rather than confirmatory generalization.

**H2:** at least one external system has lower direct-scam public-label-candidate
recall than the baseline under the shared action contract.

**H3:** no system dominates across protected-context escalation, direct-scam
public-label recall, review burden, incompatible-output rate, p50/mean latency,
and dated list-price estimate per 1K posts.

Primary metric: protected-context escalation rate with numerator, denominator,
and 10,000 fixed-seed percentile-bootstrap 95% CI. Report paired differences
from the baseline. Secondary metrics are reported with exact counts. The six
case slice is directional evidence only; it cannot establish a general stance
effect.

## 5. Costs, Provenance, and Deviations

Before v2 calls, save a dated provider-price snapshot with URLs, pin the prompt
SHA-256 in each record, cache input/output token counts and retry attempts, and
write every output atomically. Report both list-price estimate and any provider
invoice total if available; never substitute one for the other.

Any model, prompt, mapping, dataset, rate, retry, output-parser, or budget
change is a dated deviation recorded before aggregate interpretation. V2 does
not overwrite or reclassify v1 results.

## 6. Execution Gate

No v2 external call is authorized by this document. Before execution, confirm
the current provider model availability, cost cap, and the shared adapter's
unit tests. The public UI remains unchanged until the v2 result and limitations
